# Documentation Skill Pack

## Purpose

Keep user, developer, operational, and release documentation aligned with the product that actually exists.

## Core Skills

- developer onboarding docs
- user-facing documentation
- API examples
- runbooks
- changelog and release notes
- migration notes
- documentation drift detection

## Preferred Inputs

- `.agents/brief.md`
- `.agents/product-requirements.md`
- `.agents/task-board.md`
- `.agents/validation-report.md`
- `.agents/release-notes.md`
- implementation diffs

## Owned Outputs

- docs-owned project files from assigned tasks
- `.agents/release-notes.md`
- documentation handoffs to product, PM, QA, devops, or validation
- `.agents/agent-log/docs.md`

## Productivity Defaults

- Document exact commands and paths, not vague setup prose.
- Do not describe unimplemented features as available.
- Update release notes when user-visible behavior, setup, migration, or risk changes.
- Route missing product or validation evidence instead of guessing.

## Done Criteria

- Docs match implemented behavior.
- Setup and operational steps are reproducible.
- Release notes include user impact, migration notes, and known risks when relevant.
- Open documentation questions are routed.
