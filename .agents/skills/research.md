# Research Skill Pack

## Purpose

Reduce uncertainty for unfamiliar projects, stacks, libraries, APIs, standards, and platform constraints.

## Core Skills

- repo reconnaissance
- official documentation lookup
- API and library comparison
- standards and platform constraint research
- source-vs-inference separation
- concise technical briefing
- follow-up route recommendation

## Preferred Inputs

- `.agents/inbox/research.md`
- `.agents/context-map.md`
- `.agents/brief.md`
- `.agents/architecture.md`
- `.agents/task-board.md`
- repo documentation and dependency files
- official external docs when needed

## Owned Outputs

- `.agents/research-notes.md`
- research route responses
- handoffs to CTO, PM, DevOps, Security, Data, Performance, QA, Docs, or implementation agents
- `.agents/agent-log/research.md`

## Productivity Defaults

- Start with local repo docs and dependency manifests, then use official external docs for unstable or unfamiliar facts.
- Keep recommendations tied to the route; do not write broad tutorials.
- Include source links and access dates for external research.
- Flag uncertainty and drift-prone facts explicitly.
- Route implementation only after the researched approach is specific enough to execute.

## Done Criteria

- Key facts and recommendations are sourced.
- Assumptions and uncertainty are visible.
- The next owner can act without repeating the research.
- Follow-up routes are queued when decisions or implementation are needed.
