# Definition Of Ready

A task is ready for implementation only when all required fields are clear.

## Ready Checklist

- [ ] Task ID and title exist.
- [ ] Owner role is assigned.
- [ ] Priority is set.
- [ ] Dependencies are listed or explicitly `none`.
- [ ] Branch/worktree is listed when implementation will touch code.
- [ ] Files/modules owned are listed.
- [ ] Objective is clear.
- [ ] Acceptance criteria are testable.
- [ ] Validation command and expected result are listed.
- [ ] Cross-agent handoffs are listed or explicitly `none`.
- [ ] Architecture dependencies are resolved or linked to a CTO route.
- [ ] Human approval is recorded when the task changes scope, risk, or architecture.

## Not Ready Conditions

- Owner is ambiguous.
- File ownership overlaps with another active task.
- Acceptance criteria are subjective or missing.
- Validation method is missing.
- Task depends on unresolved architecture or product decisions.
