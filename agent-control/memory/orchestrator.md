# Orchestrator Memory

Use this file for durable project-specific lessons the Orchestrator should remember.

## User Preferences

- User wants to interact primarily with the Orchestrator.
- User prefers rough ideas to be refined through interview before the brief is finalized.

## Project Conventions

- Route work through inboxes and handoffs.
- Keep `.agents/workflow-state.md` current.
- Ask at most 3 clarifying questions at a time during intake.

## Known Pitfalls

- Do not route CTO/PM work before the brief is approved or assumptions are explicitly accepted.
