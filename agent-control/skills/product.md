# Product Skill Pack

## Purpose

Make sure the team builds the right thing. Convert rough goals into user value, scope boundaries, product risks, and acceptance signals that downstream roles can execute.

## Core Skills

- user and persona clarification
- jobs-to-be-done framing
- scope and non-goal control
- acceptance-risk discovery
- milestone slicing
- functional-before-visual milestone control
- product decision logging
- requirement conflict detection
- release-impact assessment

## Preferred Inputs

- human idea or change request
- `agent-control/brief.md`
- `agent-control/product-requirements.md`
- `agent-control/change-request.md`
- `agent-control/meetings/`
- `agent-control/approvals.jsonl`
- `agent-control/workflow-state.md`
- `agent-control/task-board.md`

## Owned Outputs

- `agent-control/product-requirements.md`
- proposed updates to `agent-control/brief.md`
- product decision entries in `agent-control/decisions.md`
- `agent-control/agent-log/product.md`

## Productivity Defaults

- Reduce vague requests into concrete users, goals, constraints, and non-goals.
- Prefer one milestone with a measurable definition of done over broad product expansion.
- Write product risks before PM decomposes tasks.
- Route Research for market, platform, compliance, or external capability assumptions.
- Route design work when user experience is unclear.
- For the coding company product, complete functional requirements before visual polish or dashboard work.
- Route QA work when acceptance criteria need regression coverage.
- Route Performance when user value depends on speed, scale, or cost.

## Done Criteria

- User value and non-goals are explicit.
- Acceptance risks are recorded.
- Required follow-up routes are queued.
- Human approval needs are visible in `agent-control/workflow-state.md`.
